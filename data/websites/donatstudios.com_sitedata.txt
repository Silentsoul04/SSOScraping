CSV files are frustratingly misunderstood, abused, and most of all underspecified. While <a href=\"https://tools.ietf.org/html/rfc4180\" title=\"RFC4180 Common Format and MIME Type for Comma-Separated Values (CSV) Files\">RFC4180</a> exists, it is far from definitive and goes largely ignored.</p>\n<p>Partially as a companion piece to my recent post about how CSV is an <a href=\"https://donatstudios.com/CSV-An-Encoding-Nightmare\">encoding nightmare</a>, and partially an expression of frustration, I've decided to make a list of falsehoods programmers believe about CSVs. I recommend my previous post for a more in-depth coverage on the pains of CSVs encodings and how the default tooling (Excel) will ruin your day.</p>\n<p><strong>Everything on this list is a false assumption</strong> that developers make.</p>\n<ol>\n<li>All CSVs are ASCII</li>\n<li>All CSVs are Win1252</li>\n<li>All CSVs are in 8-bit encodings</li>\n<li>All CSVs are UTF-8</li>\n<li>All CSVs are UTF-16</li>\n<li>All CSVs contains a single consistent encoding</li>\n<li>All records contain a single consistent encoding</li>\n<li>All fields contain a single consistent encoding</li>\n<li>All CSVs contain records</li>\n<li>All records contain fields </li>\n<li>Fields never contain record separators </li>\n<li>Fields never contain delimiters </li>\n<li>Fields never contain control characters</li>\n<li>Delimiters are escaped with a <code>\\</code></li>\n<li>All fields are enclosed by double quotes</li>\n<li>All records are a single line</li>\n<li>All lines contain a single record</li>\n<li>All records contain the same number of fields</li>\n<li>All records contain the same number of fields as the header</li>\n<li>All records contain the same number of fields or fewer than the header</li>\n<li>All CSVs contain a header</li>\n<li>All record separators are CRLF</li>\n<li>All record separators are LF</li>\n<li>All record separators are a single byte</li>\n<li>All record separators are a single rune</li>\n<li>All newlines are a single byte</li>\n<li>All CSVs are delimited with a comma</li>\n<li>All CSVs are delimited with a comma, tab or semicolon</li>\n<li>TSV isn't CSV</li>\n<li>All delimiters are a single byte</li>\n<li>All commas are a single byte</li>\n<li>All CSVs are readable with Excel</li>\n<li>Excel is a good tool for working with CSVs</li>\n<li>Excel is an OK tool for working with CSVs</li>\n<li>Excel can losslessly save CSVs it opens</li>\n<li>Using <code>=\"{value}\"</code> is a <em>good</em> way to get around Excel auto-formatting</li>\n<li>The first line will never be a poorly supported instruction header</li>\n<li>Using <code>sep={char}</code> is a <em>good</em> way to get Excel to accept your delimiter</li>\n<li>Prepending a BOM is a good way to get Excel to read your encoding</li>\n<li>You can safely name your first column \"ID\"</li>\n<li>All CSVs follow <a href=\"https://tools.ietf.org/html/rfc4180\" title=\"RFC4180 Common Format and MIME Type for Comma-Separated Values (CSV) Files\">RFC4180</a></li>\n<li>Most CSVs follow <a href=\"https://tools.ietf.org/html/rfc4180\" title=\"RFC4180 Common Format and MIME Type for Comma-Separated Values (CSV) Files\">RFC4180</a></li>\n<li>All CSVs follow the same defined standard</li>\n<li>All CSVs follow a defined standard</li>\n<li>All CSVs have a <code>.csv</code> extension</li>\n<li>All CSV is human readable</li>\n</ol>\n<p>Please take these into consideration next time you find yourself working with CSV. If you can think of anything I may have missed I'd be happy to add it.</p>\n<p>As a suggested further reading, \"The Art of Unix Programming\" <a
