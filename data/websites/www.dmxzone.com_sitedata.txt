where people are asking for more custom affordable webapp solutions. Just using an opensource system is not enough anymore since more customization is often needed we also need a way to achieve this in a timeless affordable way.&nbsp;</p><p style=\"text-align: center\"><img src=\"downloads/images/Bootstrap3_tips.jpg\" style=\"font-size: 10pt\" width=\"720\" height=\"240\" alt=\"\" border=\"0\" align=\"baseline\">&nbsp;</p>  </div> <div style=\"margin: 30px 0 15px 0;line-height:36px;\">    <button class=\"btn btn-small\" onclick=\"$.scrollTo('#overview');\"><i class=\"icon\" style=\"width:2px;\"></i>The Story</button>       <button class=\"btn btn-small\" onclick=\"$.scrollTo('#extensions');\"><i class=\"icon\" style=\"width:2px;\"></i>Extensions</button>             <button class=\"btn btn-small\" onclick=\"$.scrollTo('#tips');\"><i class=\"icon\" style=\"width:2px;\"></i>Tips</button>   <button class=\"btn btn-small\" onclick=\"$.scrollTo('#comments');\"><i class=\"icon\"><small>0</small></i>Comments</button>  </div> </div> </div>       <div class=\"item\"> <div id=\"the-story\" class=\"subarticle\"> <div class=\"pull-right\"><a href=\"javascript:void(0);\" onclick=\"$.scrollTo(0);\">Back to Top</a></div> <div class=\"article-subtitle border-color-showcases\">The Story</div> <div class=\"article-body\">  <p><span style=\"font-size: 13.3333px\">I think that we are in the moment where people are heading toward more custom needs, that can transform their business working model into a meaningful website/webapp. This is where DMXzone gives some amazing opportunities to create high quality custom stuff while focusing on creativity. I personally think that DMXzone offers you a front-end &amp; a back-end developer transforming you into a project manager. I have been working with their tools for many years and I think that it is a matter of time that people will get more and more developing with their tools. Always have a plan before developing</h4>  <p>Always have a draft of your layout on paper or on your screen, where you will be able to see how your layout is organized in containers and rows. So, for example, this layout </p>  <p class=\"align-center\">  <a title=\"Always have a plan before developing\" rel=\"dmxLightbox\" rev=\"{preset:'dmxzone',group:'features'}\" href=\"/downloads/media/32849/01separateAreas.jpg\"> <img src=\"/res/jpg/w690/downloads/media/32849/01separateAreas.jpg\" style=\"max-width:100%;\"> </a>  </p>     <h4 id=\"the-container-with-2-rows-and-some-columns\" style=\"position:relative;margin-top:30px;\">2. The container, with 2 rows and some columns</h4>  <p>So if you just separate your layout in 'Bootstrap' &lt;div&gt; (containers, rows, columns) it is then much easier to create almost anything\nWhy you should do this: It is much easier to detect which 'Bootstrap' &lt;div&gt; to use if you outline the areas of your layout.\n</p>  <p class=\"align-center\">  <a title=\"The container, with 2 rows and some columns\" rel=\"dmxLightbox\" rev=\"{preset:'dmxzone',group:'features'}\" href=\"/downloads/media/32849/02separateAreas.jpg\"> <img src=\"/res/jpg/w690/downloads/media/32849/02separateAreas.jpg\" style=\"max-width:100%;\"> </a>  </p>     <h4 id=\"use-a-page-as-a-starter-point-for-all-of-your-changes\" style=\"position:relative;margin-top:30px;\">3. Use a page as a starter point for all of your changes</h4>  <p>For example I create a page called _blank.html or _starter.html and in there apply all additional layout changes. Then, when happy with the result, just copy these changes on the rest of your pages. \nWhy you should do this. Well, sometimes when some of your developing pages are full with other dynamic data, is easy to mess up. Having a _starter page is easier to apply changes and then move them to any page needed.\n</p>  <p class=\"align-center\">  <a title=\"Use a page as a starter point for all of your changes\" rel=\"dmxLightbox\" rev=\"{preset:'dmxzone',group:'features'}\" href=\"/downloads/media/32849/02_02_starterpage.jpg\"> <img src=\"/res/jpg/w690/downloads/media/32849/02_02_starterpage.jpg\" style=\"max-width:100%;\"> </a>  </p>     <h4 id=\"comment-your-code\" style=\"position:relative;margin-top:30px;\">4. Comment your code</h4>  <p>When adding containers, rows, columns it easy to get lost.Always comment some blocks of your layout. For example you add a class=”row” where it will be used for products display. You can use this &lt;!—products—&gt; at the start and &lt;!—//newProducts—&gt; at the end of this “row”. In fact what I do is place the comment like this &lt;!——————————products—————————&gt; so it is more easy to locate.  \nWhy you should do this: Just do it. You will understand why, when you will need to make any changes a few days later.\n</p>  <p class=\"align-center\">  <a title=\"Comment your code\" rel=\"dmxLightbox\" rev=\"{preset:'dmxzone',group:'features'}\" href=\"/downloads/media/32849/03_01_comments.jpg\"> <img src=\"/res/jpg/w690/downloads/media/32849/03_01_comments.jpg\" style=\"max-width:100%;\"> </a>  </p>     <h4 id=\"property-inspector-not-updating\" style=\"position:relative;margin-top:30px;\">5. Property inspector not updating</h4>  <p>Sometimes when the properties inspector won’t show your latest changes made on your layout just save the page and it will update If DW can't detect the DOM in the property inspector just save the page.\nWhy you should do this: It is always safer to select any blocks of code through the property inspector. This way you will be sure that DW selected the block you want to. Handcoding in DW sometimes does not refresh the property inspector. A simple 'save' will do the job.</p>  <p class=\"align-center\">  <a title=\"Property inspector not updating\" rel=\"dmxLightbox\" rev=\"{preset:'dmxzone',group:'features'}\" href=\"/downloads/media/32849/04_01_properties_inspector.jpg\"> <img src=\"/res/jpg/w690/downloads/media/32849/04_01_properties_inspector.jpg\" style=\"max-width:100%;\"> </a>  </p>     <h4 id=\"add-new-id-or-class-for-additional-styling-and-lt-style-and-gt-changes\" style=\"position:relative;margin-top:30px;\">6. Add new 'id' or 'class' for additional styling &lt;style&gt; changes</h4>  <p>If you want to create some additional styling just add a new class or id and style it in an external css file other than the one BS3 is creating. What I often do is to create an inline styling just above the block I want to style and when happy with the result I move this &lt;styling&gt; block in an external .css file\nWhy you should do this: I know it is tempting to have as less files as possible but, don't mess your code with existing files created by an extension. If, for some reason, you make an update, from the extension panel, you will lose all of your changes. Instead add an additional 'id' or 'class' to the element you want to style and style it in a new .css file</p>  <p class=\"align-center\">  <a title=\"Add new 'id' or 'class' for additional styling <style> changes\" rel=\"dmxLightbox\" rev=\"{preset:'dmxzone',group:'features'}\" href=\"/downloads/media/32849/05_01_newStyling.jpg\"> <img src=\"/res/jpg/w690/downloads/media/32849/05_01_newStyling.jpg\" style=\"max-width:100%;\"> </a>  </p>     <h4 id=\"use-some-other-bootstrap-3-components-along-with-the-dmxzone-bootstrap-3\" style=\"position:relative;margin-top:30px;\">7. Use some other Bootstrap 3 components along with the DMXzone Bootstrap 3</h4>  <p>Just integrate the new component (slider etc) in a starter layout and when working properly apply changes on your actual pages.\nWhy you should do this: Don't reinvent the wheel. If you have any code snippets that work perfect for you, just use them. Leave more place for creative time, rather than trying to build something that is already perfect for what you want to achieve. </p>  <p class=\"align-center\">  <a title=\"Use some other Bootstrap 3 components along with the DMXzone Bootstrap 3\" rel=\"dmxLightbox\" rev=\"{preset:'dmxzone',group:'features'}\" href=\"/downloads/media/32849/06_01external.png\"> <img src=\"/res/jpg/w690/downloads/media/32849/06_01external.png\" style=\"max-width:100%;\"> </a>  </p>    </div> </div> </div>  <div class=\"item\"> <div class=\"subarticle\"> <div
