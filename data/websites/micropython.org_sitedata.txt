and highly robust </h1>\n  </div>\n  <div class=\"col-md-offset-2 col-md-8\">\n    <p>\n    MicroPython employs many advanced coding techniques, and lots of tricks\n    to maintain a compact size while still having a full set of features.\n    </p>\n    <p>\n    Some of the more notable items are:\n    </p><ul>\n      <li> highly configurable due to many compile-time configuration options </li>\n      <li> support for many architectures (x86, x86-64, ARM, ARM Thumb, Xtensa)</li>\n      <li> extensive <a href=\"https://github.com/micropython/micropython/tree/master/tests\">test suite</a> with over 590 tests, and more than 18,500 individual testcases </li>\n      <li> code coverage at 98.4% for the core and <a href=\"https://coveralls.io/github/micropython/micropython\">at 96.3%</a> for the core plus extended modules </li>\n      <li> fast start-up time from boot to loading of first script (150 microseconds to get to boot.py, on PYBv1.1 running at 168MHz) </li>\n      <li> a simple, fast and robust mark-sweep garbage collector for heap memory </li>\n      <li> a MemoryError exception is raised if the heap is exhausted </li>\n      <li> a RuntimeError exception is raised if the stack limit is reached </li>\n      <li> support for running Python code on a hard interrupt with minimal latency </li>\n      <li> errors have a backtrace and report the line number of the source code </li>\n      <li> constant folding in the parser/compiler </li>\n      <li> <a href=\"https://github.com/micropython/micropython/blob/master/py/mpconfig.h#L51-L95\">pointer tagging</a> to fit small integers, strings and objects in a machine word </li>\n      <li> transparent transition from small integers to big integers </li>\n      <li> support for 64-bit NaN boxing object model </li>\n      <li> support for 30-bit stuffed floats, which don't require heap memory  </li>\n      <li> a cross-compiler and frozen bytecode, to have pre-compiled scripts that don't take any RAM (except for any dynamic objects they create) </li>\n      <li> multithreading via the \"_thread\" module, with an optional global-interpreter-lock (still work in progress, only available on selected ports) </li>\n      <li> a native emitter that targets machine code directly rather than the bytecode virtual machine</li>\n      <li> inline assembler (currently Thumb and Xtensa instruction sets only) </li>\n    </ul>\n    <p></p>\n  </div>\n</div>\n\n<div class=\"row row-white\">\n  <div class=\"col-md-12 text-center\">\n    <h1> Online resources </h1>\n  </div>\n  <div
