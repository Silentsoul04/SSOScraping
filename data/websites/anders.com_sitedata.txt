href=\"/cms/459/Mac/Create/Fusion.Drive/Terminal\">Create a Fusion Drive in Mac OS X</a></h1>\n        <p class=\"meta\">\n         <time datetime=\"2016-02-05 17:05:47+01:00\" pubdate=\"\" data-updated=\"true\">February 5, 2016</time>\n        </p>\n       </header>\n\n       <div class=\"entry-content\">\n        <p>Warning: You must watch what you are doing here / backup appropriately  - Terminal will happily delete everything on you. You have been warned.<br>\n<br>\nMac Fusion drives are logical volumes created by combining \"spinning platter\" and SSD drives in software. Here's how you make them using diskutil in Terminal:<br>\n<br>\nSee all your volumes:<br>\n<br>\ndiskutil -list<br>\n<br>\nMake note of the volumes you intend to use. In my case that was /dev/disk0 and /dev/disk1 because I was recreating my primary drive into a fusion drive. (unless you are doing exactly the same thing, use different values as /dev/disk0 is usually your boot drive. You have been warned!)<br>\n<br>\nCreate a logical volume group:<br>\n<br>\ndiskutil cs create lvg0 /dev/drive0 /dev/drive1<br>\n<br>\nCopy the resulting UUID - you'll need it in the next step.<br>\n<br>\nCreate the logical volume:<br>\n<br>\ndiskutil cs createVolume de305d54-75b4-431b-adb2-eb6b9e546014 \"Case-sensitive Journaled HFS+\" \"Fusion Drive\" 100%<br>\n<br>\nThat's it! Now you should have a case sensitive journaled HFS+ formatted partition covering your entire fusion drive.</p>\n       </div>\n\n       <div class=\"entry-content\">\n        <p><a href=\"/cms/459/Mac/Create/Fusion.Drive/Terminal#comments\">Comments</a></p>\n       </div>\n\n       <footer>\n        <p class=\"meta\">\n         <span class=\"byline author vcard\">Posted by <span class=\"fn\"><a rel=\"author\" href=\"http://andersbrownworth.com/\">Anders Brownworth</a></span></span>\n         <time datetime=\"2016-02-05 17:05:47+01:00\" pubdate=\"\" data-updated=\"true\">February 5, 2016</time>\n        </p>\n\n        <div class=\"sharing\">\n         <a href=\"http://twitter.com/share\" class=\"twitter-share-button\" data-url=\"http://anders.com/cms/459/Mac/Create/Fusion.Drive/Terminal\" data-via=\"Anders94\" data-counturl=\"http://anders.com/cms/459/Mac/Create/Fusion.Drive/Terminal\">Tweet</a>\n         <div class=\"g-plusone\" data-size=\"medium\"></div>\n         <div class=\"fb-like\" data-send=\"true\" data-width=\"450\" data-show-faces=\"false\"></div>\n        </div>\n       </footer>\n      </article>\n\n      <article class=\"hentry\" role=\"article\">\n\n       <header>\n        <h1 class=\"entry-title\"><a href=\"/cms/458/postgresql/schema/function/extension\">Postgres Extensions and Functions in Schemas</a></h1>\n        <p class=\"meta\">\n         <time datetime=\"2016-02-03 13:04:00+01:00\" pubdate=\"\" data-updated=\"true\">February 3, 2016</time>\n        </p>\n       </header>\n\n       <div class=\"entry-content\">\n        <p>Adding an extension to PostgreSQL usually implicitly sets up some functions enabling use of the extension. The extensions are global but the functions created to make use of the extension are schema specific. Therefore, if you do this:<br>\n<br>\nSET search_path = reports;<br>\nCREATE EXTENSION \"uuid-ossp\";<br>\n<br>\nyou are implicitly creating these functions in the \"reports\" schema:<br>\n<br>\nuuid_generate_v1()<br>\nuuid_generate_v1mc()<br>\nuuid_generate_v3()<br>\nuuid_generate_v4()<br>\nuuid_generate_v5()<br>\nuuid_nil()<br>\nuuid_ns_dns()<br>\nuuid_ns_oid()<br>\nuuid_ns_url()<br>\nuuid_ns_x500()<br>\n<br>\nYou can see what functions are within a particular schema in psql using:<br>\n<br>\n\\df reports.*<br>\n<br>\nIf you need to use these functions in other schemas, you have to either fully qualify the function:<br>\n<br>\nSELECT reports.uuid_generate_v4();<br>\n<br>\nor you have to get a copy of the function which uses the global extension in the schema you are in:<br>\n<br>\ndb=# set search_path=marketing;<br>\nSET<br>\ndb=# CREATE FUNCTION uuid_v4()<br>\ndb-# RETURNS uuid<br>\ndb-# LANGUAGE c<br>\ndb-# STRICT<br>\ndb-# AS '$libdir/uuid-ossp', $function$uuid_generate_v4$function$;<br>\nCREATE FUNCTION<br>\ndb=# SELECT marketing.uuid_v4();<br>\n               uuid_v4<br>\n--------------------------------------<br>\n 33675b19-aa8b-41b5-b07c-06e3f774e588<br>\n(1 row)<br>\n<br>\nAnother way to go would be to create a schema called \"extensions\" and do your extension install there which would make all your functions live there too. Then anywhere you needed to use an extension, you would qualify it with an \"extensions.\" prefix.<br>\n<br>\nCREATE TABLE foo (<br>\n  id  uuid  DEFAULT extensions.uuid_generate_v4()<br>\n);</p>\n       </div>\n\n       <div class=\"entry-content\">\n        <p><a href=\"/cms/458/postgresql/schema/function/extension#comments\">Comments</a></p>\n       </div>\n\n       <footer>\n        <p class=\"meta\">\n         <span class=\"byline author vcard\">Posted by <span class=\"fn\"><a rel=\"author\" href=\"http://andersbrownworth.com/\">Anders Brownworth</a></span></span>\n         <time datetime=\"2016-02-03 13:04:00+01:00\" pubdate=\"\" data-updated=\"true\">February 3, 2016</time>\n        </p>\n\n        <div class=\"sharing\">\n         <a href=\"http://twitter.com/share\" class=\"twitter-share-button\" data-url=\"http://anders.com/cms/458/postgresql/schema/function/extension\" data-via=\"Anders94\" data-counturl=\"http://anders.com/cms/458/postgresql/schema/function/extension\">Tweet</a>\n         <div class=\"g-plusone\" data-size=\"medium\"></div>\n         <div class=\"fb-like\" data-send=\"true\" data-width=\"450\" data-show-faces=\"false\"></div>\n        </div>\n       </footer>\n      </article>\n\n      <article class=\"hentry\" role=\"article\">\n\n       <header>\n        <h1 class=\"entry-title\"><a href=\"/cms/457/PostgreSQL/high.row.count\">600+M Rows in PostgreSQL</a></h1>\n        <p class=\"meta\">\n         <time datetime=\"2015-07-25 12:01:24+01:00\" pubdate=\"\" data-updated=\"true\">July 25, 2015</time>\n        </p>\n       </header>\n\n       <div class=\"entry-content\">\n        <p>I've been working on a PostgreSQL project which has a table with 644,742,830 (or 644.7M) rows in it. There is an index on an array column, in this case a column of type character varying(35)[] with a UNIQUE constraint. Each row has exactly two strings in it which happen to be bitcoin addresses. When I SELECT some address from that column:<br>\n<br>\nSELECT * FROM keys WHERE addresses @&gt; '1Axrv5WPGA65wpxCpV87sYdwxhkmy8mvF2';<br>\n<br>\nthe database is actually SELECTing from amongst 1,289,485,660 (or 1.2 Billion) keys. Because the database is on an SSD and there is enough RAM in the system to fit the index, I can regularly return results within the 1ms to 2ms range!<br>\n<br>\nOn disk, the entire database (including a few other columns and several other tables) is in the 421Gig range. I had been worrying about having to use some other strategy (LevelDB possibly) but have been amazed by the results. I'll keep adding rows until I ether run out of disk space or run into a performance problem. Gotta love good indexes!</p>\n       </div>\n\n       <div class=\"entry-content\">\n        <p><a href=\"/cms/457/PostgreSQL/high.row.count#comments\">Comments</a></p>\n       </div>\n\n class=\"entry-title\"><a href=\"/cms/456/postgres/uuid/operator.class/index\">PostgreSQL Index Type for UUID[] Arrays</a></h1>\n        <p class=\"meta\">\n         <time datetime=\"2015-02-12 08:22:00+01:00\" pubdate=\"\" data-updated=\"true\">February 12, 2015</time>\n        </p>\n       </header>\n\n       <div class=\"entry-content\">\n        <p>PostgreSQL doesn't include an index type for uuid array columns.<br>\n<br>\nHere's a table with a uuid array column:<br>\n<br>\n---------------------------------------------------<br>\nCREATE TABLE items (<br>\n  things uuid[]<br>\n);<br>\n---------------------------------------------------<br>\n<br>\nBut when we try to create an index on it:<br>\n<br>\n---------------------------------------------------<br>\nCREATE INDEX items_things_idx ON items USING GIN(things);<br>\nERROR:  data type uuid[] has no default operator class for access method \"gin\"<br>\n---------------------------------------------------<br>\n<br>\nSo we must create an index type that understands how to compare elements in an array of UUIDs:<br>\n<br>\n---------------------------------------------------<br>\nCREATE OPERATOR CLASS _uuid_ops DEFAULT FOR TYPE _uuid USING gin AS<br>\n      OPERATOR 1 &amp;&amp;(anyarray, anyarray),<br>\n      OPERATOR 2 @&gt;(anyarray, anyarray),<br>\n      OPERATOR 3 &lt;@(anyarray, anyarray),<br>\n      OPERATOR 4 =(anyarray, anyarray),<br>\n      FUNCTION 1 uuid_cmp(uuid, uuid),<br>\n      FUNCTION 2 ginarrayextract(anyarray, internal, internal),<br>\n      FUNCTION 3 ginqueryarrayextract(anyarray, internal, smallint, internal, internal, internal, internal),<br>\n      FUNCTION 4 ginarrayconsistent(internal, smallint, anyarray, integer, internal, internal, internal, internal),<br>\n      STORAGE uuid;<br>\n---------------------------------------------------<br>\n<br>\nNow our index will create and SELECTs to items in the UUID array will be fast. For example, find all the rows that contain a particular UUID anywhere in the array:<br>\n<br>\n---------------------------------------------------<br>\nSELECT * FROM items WHERE things @&gt; ARRAY['a6e34e5d-b1fb-4240-8ad9-21ddf23134bb']::uuid[];<br>\n---------------------------------------------------<br>\n<br>\nThere's a <a href=\"https://coderwall.com/p/1b5eyq/index-for-uuid-array-data-type\">better-formatted version of this</a>.</p>\n       </div>\n\n       <div class=\"entry-content\">\n        <p><a
