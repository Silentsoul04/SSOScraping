news...</a></p>\n</div>\n\n<h2>Overview</h2>\n\n<p>Dovecot is an <a href=\"doc/COPYING\">open source</a>\n<a href=\"http://en.wikipedia.org/wiki/Imap\">IMAP</a> and\n<a href=\"http://en.wikipedia.org/wiki/Pop3\">POP3</a> email server\nfor Linux/UNIX-like systems, written with security primarily in mind.\nDovecot is an excellent choice for both small and large installations. It's\nfast, simple to set up, requires no special administration and it uses very\nlittle memory.</p>\n\n<p>Some of the most notable features of Dovecot include:</p>\n\n<ul>\n  <li>Dovecot is among the <strong>highest performing</strong> IMAP servers\n   while still supporting the <strong>standard</strong>\n   <a href=\"http://wiki2.dovecot.org/MailboxFormat/mbox\">mbox</a> and\n   <a href=\"http://wiki2.dovecot.org/MailboxFormat/Maildir\">Maildir</a>\n   formats. The mailboxes are transparently indexed, which gives Dovecot its\n   good performance while still providing full compatibility with existing\n   mailbox handling tools.</li>\n\n  <li>Dovecot is <strong>standards compliant</strong>. Dovecot v1.1 passes\n   all <a href=\"http://imapwiki.org/ImapTest/ServerStatus\">IMAP server\n   standard compliancy tests</a> while most other servers fail many of\n   them.</li>\n\n  <li>Dovecot's indexes are <strong>self-optimizing</strong>. They contain\n   exactly what the user's client commonly needs, no more and no less.</li>\n\n  <li>Dovecot is <strong>self-healing</strong>. It tries to fix most of the\n   problems it notices by itself, such as broken index files. The problems are\n   however logged so the administrator can later try to figure out what caused\n   them.</li>\n\n  <li>Dovecot tries to be <strong>admin-friendly</strong>. Common error\n   messages are made as easily understandable as possible. Any crash, no\n   matter how it happened, is considered a bug that will be fixed.</li>\n\n  <li>Dovecot allows mailboxes and their indexes to be modified by multiple\n   computers at the same time, while still performing well. This means that\n   Dovecot <strong>works well with clustered filesystems</strong>.\n   <a href=\"http://wiki2.dovecot.org/NFS\">NFS has caching problems</a>,\n   but you can work around them with <a href=\"http://wiki2.dovecot.org/Director\">\n   director proxies</a>.</li>\n\n  <li>Dovecot's user authentication is extremely flexible and feature-rich,\n   supporting many different\n   <a href=\"http://wiki2.dovecot.org/Authentication\">authentication\n   databases</a> and\n   <a href=\"http://wiki2.dovecot.org/Authentication/Mechanisms\">mechanisms.</a>\n   </li>\n\n  <li><a href=\"http://www.postfix.org/\">Postfix</a> 2.3+ and\n   <a href=\"http://www.exim.org/\">Exim</a> 4.64+ users can do SMTP\n   authentication directly against Dovecot's authentication backend without\n   having to configure it separately.</li>\n\n  <li>Dovecot supports easy\n   <a href=\"http://wiki2.dovecot.org/Migration\">migration from many existing\n   IMAP and POP3 servers</a>, allowing the change to be transparent to\n   existing users.</li>\n\n  <li>Dovecot supports <a href=\"http://wiki2.dovecot.org/Clients\">workarounds</a>\n   for several bugs in IMAP and POP3 clients. Since the workarounds may cause\n   the protocol exchange to be suboptimal, you can enable only the\n   workarounds you need.</li>\n\n  <li>Dovecot's <a href=\"http://wiki2.dovecot.org/Design/Processes\">design</a>\n   and <a href=\"/doc/securecoding.txt\">implementation</a> is highly focused on\n   security. Rather than taking the traditional road of just fixing\n   vulnerabilities whenever someone happens to report them,\n   <a href=\"security.html\">I offer 1000 EUR</a> of my own money to the first\n   person to find a security hole from Dovecot.</li>\n\n  <li>Dovecot is easily extensible.\n   <a href=\"http://wiki2.dovecot.org/Plugins\">Plugins</a> can add new commands,\n   modify existing behavior, add their own data into index files or even add\n   support for new mailbox formats. For example\n   <a href=\"http://wiki2.dovecot.org/Quota\">quota</a> and\n   <a href=\"http://wiki2.dovecot.org/ACL\">ACL support</a> are completely\n   implemented as plugins.</li>\n</ul>\n\n<h2>Status</h2>\n\n<ul>\n  <li>Complete IMAP4rev1 and POP3 support. IPv6, SSL and TLS are supported.</li>\n  <li>Supports multiple commonly used IMAP extensions, including SORT,\n      THREAD and IDLE.</li>\n  <li>Shared mailboxes are fully supported in v1.2+. Older versions also\n      support admin-configurable\n      <a href=\"http://wiki2.dovecot.org/ACL\">ACL files</a>.</li>\n  <li><a href=\"http://wiki2.dovecot.org/Quota/Maildir\">Maildir++ quota</a> is\n      supported, but <a href=\"http://wiki2.dovecot.org/Quota/FS\">hard\n      filesystem quota</a> can be problematic.</li>\n  <li>Dovecot is commonly used with Linux, Solaris, FreeBSD, OpenBSD, NetBSD\n      and Mac OS X.
