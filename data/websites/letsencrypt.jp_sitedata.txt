総合ポータル」がオープン</dd>\n\n\t\t\t\t</dl>\n\t\t\t</section>\n\n\t\t\t<section>\n\t\t\t\t<div class=\"lastUpdate\"><time datetime=\"2016-06-08\">2016年06月08日</time> 更新</div>\n\t\t\t\t<h2 id=\"CompatibleBrowser\">Let's Encrypt の対応ブラウザ</h2>\n\t\t\t\t<p>Let's Encrypt の「SSL/TLSサーバ証明書」は、アメリカ合衆国大手認証局（<abbr title=\"Certificate Authority\">CA</abbr>）である IdenTrust 社のルート証明書からチェーンできるクロスルート証明書です。IdenTrust 社の証明書（DST Root CA X3）によって、中間証明書「Let's Encrypt Authority X3」、および予備の中間証明書「Let's Encrypt Authority X4」に対するクロス署名が行われています。</p>\n\t\t\t\t<p>そのため、Let's Encrypt 発行の証明書は、大手認証局（<abbr title=\"Certificate Authority\">CA</abbr>）が有償で販売している証明書と同様に、ほとんどのWebブラウザやOSが標準で対応しています。</p>\n\t\t\t\t<h4>主な対応ブラウザとOS</h4>\n\t\t\t\t<ul>\n\t\t\t\t\t<li>Microsoft Internet Explorer 6 以上 (Windows XP SP3 以上<span class=\"sup\">※1</span>、Server 2003 SP2 以上<span class=\"sup\">※2</span>)<span class=\"sup\">※3</span></li>\n\t\t\t\t\t<li>Microsoft Edge (Windows 10)</li>\n\t\t\t\t\t<li>Google Chrome (Windows XP SP3 以上<span class=\"sup\">※1</span>、Server 2003 SP2 以上<span class=\"sup\">※2</span>、OS X 10.4 以上、Linux、Android 2.3.6 以上、iOS 3.1 以上)</li>\n\t\t\t\t\t<li>Mozilla Firefox 2.0 以上 (Windows、OS X、Linux、Android、iOS、Firefox OS)<span class=\"sup\">※4</span></li>\n\t\t\t\t\t<li>Apple Safari 4.0 以上 (OS X 10.4 以上、iOS 3.1 以上)</li>\n\t\t\t\t\t<li>Android ブラウザ (Android 2.3.6 以上)</li>\n\t\t\t\t</ul>\n\t\t\t\t<p class=\"remarks annotation\">※1 Windows XP SP3 / Vista / 7 / 8 / 8.1 / 10 に対応（<a href=\"https://letsencrypt.jp/faq/#Browser\">Windows XP 対応についての詳細情報</a>）。</p>\n\t\t\t\t<p class=\"remarks annotation\">※2 Windows Server 2003 SP2 / 2003 R2 SP2 / 2008 / 2008 R2 / 2012 / 2012 R2 / 2016 に対応。</p>\n\t\t\t\t<p class=\"remarks annotation\">\n\t\t\t\t\t※3 Windows 98 / ME / XP SP2 以下 / NT 3.51～4.0 上の <abbr title=\"Microsoft Internet Explorer 6\">IE 6</abbr> は非対応。<br>\n\t\t\t\t\t<abbr title=\"Microsoft Internet Explorer 6\">IE 6</abbr> のデフォルト設定では SSL 2.0 / SSL 3.0 のみが有効で、TLS 1.0 は無効です（ブラウザの設定変更で有効化可能）。ウェブサーバ側で SSL 3.0 を有効化することは、セキュリティ上の観点から推奨されません。\n\t\t\t\t</p>\n\t\t\t\t<p class=\"remarks annotation\">※4 Firefox が動作すれば OS のバージョンに関係なく対応しています（証明書ストアが Mozilla 独自のため）。</p>\n\t\t\t\t<p class=\"remarks\">※詳しくは <a href=\"https://letsencrypt.org/docs/certificate-compatibility/\">Certificate Compatibility (英文)</a> をご覧ください。</p>\n\t\t\t</section>\n\n\t\t\t<section>\n\t\t\t\t<h2 id=\"ValidationType\">取得可能な SSL/TLS証明書の種類</h2>\n\t\t\t\t<p>Let's Encrypt では「ドメイン認証（<abbr title=\"Domain Validation\">DV</abbr>）SSL/TLS証明書」を無料で取得することができます。</p>\n\t\t\t\t<p>「企業認証（<abbr title=\"Organization Validation\">OV</abbr>）証明書」や「<abbr title=\"Extended Validation\">EV</abbr>証明書」は、Let's Encrypt では取得できません。それらの証明書を必要とする場合には、他の認証局やその代理店から有償で購入する必要があります。</p>\n\t\t\t\t<p>それぞれの「SSL/TLSサーバ証明書」の違いは、下記の通りです。</p>\n\t\t\t\t<h3>ドメイン認証（DV : Domain Validation）証明書</h3>\n\t\t\t\t<p>ドメインの所有者であることを確認して発行するSSL/TLSサーバ証明書です。</p>\n\t\t\t\t<h3>企業認証（OV : Organization Validation）証明書</h3>\n\t\t\t\t<p>企業や組織の実在性を、第三者データベースや電話などのインターネットを経由しない方法で認証しているSSL/TLSサーバ証明書です。</p>\n\t\t\t\t<p>主にECサイトなどの商用サイトで使用されています。</p>\n\t\t\t\t<h3>EV（Extended Validation）証明書</h3>\n\t\t\t\t<p>企業の実在性を監査済みの厳格な方法を用いて確認するSSL/TLS証明書です。多くのWebブラウザで、アドレスバーの一部が緑色になり、アドレスバー上にWebサイトの運営組織名が表示されます。</p>\n\t\t\t\t<p>主に銀行や証券会社などの高いセキュリティが要求される商用サイトで使用されています。</p>\n\t\t\t</section>\n\n\t\t\t<section>\n\t\t\t\t<h2 id=\"Specification\">当サイトにおける SSL/TLS の仕様と設定について</h2>\n\t\t\t\t<p>当サイトは HTTPS 接続に Let's Encrypt の SSL/TLS サーバ証明書を使用しています。</p>\n\t\t\t\t<p>このページが表示されている場合、接続時に証明書に関するエラーが表示された場合を除き、Let's Encrypt 発行の証明書による TLS 暗号化通信が正常に機能しています。</p>\n\t\t\t\t<p>当サイトの SSL/TLS の仕様は、下記の通りです。ただし、各項目の組み合わせには制約があります。</p>\n\t\t\t\t<h3>SSL/TLS のバージョン</h3>\n\t\t\t\t<p>\n\t\t\t\t\t「TLS 1.0」「TLS 1.1」「TLS 1.2」に対応しています。<br>\n\t\t\t\t\t仕様上の脆弱性がある「SSL 2.0」と「SSL 3.0」は無効にしています。\n\t\t\t\t</p>\n\t\t\t\t<h3>認証・鍵共有における署名アルゴリズム</h3>\n\t\t\t\t<p>以下の署名アルゴリズムに対応しています。</p>\n\t\t\t\t<ul>\n\t\t\t\t\t<li>ECDHE-RSA (Forward Secrecy)</li>\n\t\t\t\t\t<li>ECDHE-ECDSA (Forward Secrecy)</li>\n\t\t\t\t\t<li>DHE-RSA (Forward Secrecy)</li>\n\t\t\t\t\t<li>DHE-DSS (Forward Secrecy)</li>\n\t\t\t\t\t<li>RSA</li>\n\t\t\t\t</ul>\n\t\t\t\t<h3>共通鍵暗号の暗号化アルゴリズム</h3>\n\t\t\t\t<p>以下のブロック暗号に対応しています。ストリーム暗号「RC4」は、暗号強度が弱いため無効にしています。</p>\n\t\t\t\t<ul>\n\t\t\t\t\t<li>AES256-GCM (AES, 鍵長・暗号強度256ビット, GCMモード)</li>\n\t\t\t\t\t<li>AES128-GCM (AES, 鍵長・暗号強度128ビット, GCMモード)</li>\n\t\t\t\t\t<li>AES256 (AES, 鍵長・暗号強度256ビット, CBCモード)</li>\n\t\t\t\t\t<li>AES128 (AES, 鍵長・暗号強度128ビット, CBCモード)</li>\n\t\t\t\t\t<li>CAMELLIA256 (CAMELLIA, 鍵長・暗号強度256ビット, CBCモード)</li>\n\t\t\t\t\t<li>CAMELLIA128 (CAMELLIA, 鍵長・暗号強度128ビット, CBCモード)</li>\n\t\t\t\t\t<li>CBC3 (3DES, 鍵長168ビット・暗号強度112ビット, CBCモード)</li>\n\t\t\t\t</ul>\n\t\t\t\t<h3>改竄検出アルゴリズム</h3>\n\t\t\t\t<p>以下の改竄検出アルゴリズム（暗号学的ハッシュ関数）に対応しています。</p>\n\t\t\t\t<ul>\n\t\t\t\t\t<li>HMAC-SHA384 (SHA-2, ダイジェスト長384ビット, TLS 1.2 のみ)</li>\n\t\t\t\t\t<li>HMAC-SHA256 (SHA-2, ダイジェスト長256ビット, TLS 1.2 のみ)</li>\n\t\t\t\t\t<li>HMAC-SHA1 (SHA-1, ダイジェスト長160ビット)</li>\n\t\t\t\t</ul>\n\t\t\t\t<p>上記の仕様は、当サイトのWebサーバ（Apache）の設定によるものであり、Let's Encrypt の SSL/TLS サーバ証明書の仕様ではありません。</p>\n\t\t\t\t<p>Let's Encrypt の SSL/TLS サーバ証明書を導入したサーバの管理者は、「Apache」「nginx」「IIS」などのWebサーバソフトウェアの設定を変更することで、「SSL/TLS のバージョン」「共通鍵暗号の暗号化アルゴリズム」「改竄検出アルゴリズム」を自由に決めることができます（他社の SSL/TLS サーバ証明書でも同様です）。</p>\n\t\t\t\t<p class=\"remarks\">※Let's Encrypt 発行のサーバ証明書の署名ハッシュアルゴリズムは SHA-2（SHA-256）に固定されており、変更できません。</p>\n\t\t\t</section>\n\n\t\t\t<section>\n\t\t\t\t<h2 id=\"SSL-TLS\">SSL/TLS という表記について</h2>\n\t\t\t\t<p>SSL は Secure Sockets Layer の略で、TLS は Transport Layer Security の略です。</p>\n\t\t\t\t<p>多くのWebサイトやメディアが TLS（TLS 1.0 以上）にも SSL という名称を用いており、TLS という名称に馴染みのない方がいらっしゃるため、当サイトでは SSL/TLS と併記しています。</p>\n\t\t\t\t<p>プロトコルとしての SSL（SSL 2.0、SSL 3.0）には仕様上の脆弱性がありますので、Apache や nginx などのWebサーバの設定を行う際には、SSL（SSL 2.0、SSL 3.0）での接続は禁止して、TLS（TLS 1.0、TLS 1.1、TLS 1.2）のみに対応させることを推奨します。</p>\n\t\t\t</section>\n\n<!-- asDoubleRectangle.inc ここから -->\n\n<dl
