class=\"post-content\">\n\t\t\t\t\t<p>When you remove a package (such as <code>sudo apt remove php5.5-cgi</code>), all files added by the package will be deleted except for user configuration files that were modified. When you purge a package (such as <code>sudo apt purge php5.5-cgi</code>), all the package files, including modified configuration files, are deleted. After a few years, your Ubuntu installation may start to accumulate old packages that were removed but not purged. Fortunately, it is easy to find those unpurged packages and purge them.</p>\n<p>The <code>dpkg</code> command can provide us with a list of removed, unpurged packages:</p>\n<pre class=\"terminal notranslate\"><span class=\"regular-prompt\">[<span class=\"user\">chris@home</span> <span class=\"path\">~</span>]$</span> <span class=\"user-input\">dpkg --list | grep ^rc</span>\nrc  php5.5-cgi       5.5.38-4+deb.sury.org~xenial+1  amd64  server-side, HTML-embedded scripting language (CGI binary)\nrc  php5.5-cli       5.5.38-4+deb.sury.org~xenial+1  amd64  command-line interpreter for the PHP scripting language\nrc  php5.5-common    5.5.38-4+deb.sury.org~xenial+1  amd64  documentation, examples and common module for PHP\nrc  php5.5-json      5.5.38-4+deb.sury.org~xenial+1  amd64  JSON module for PHP\nrc  php5.5-opcache   5.5.38-4+deb.sury.org~xenial+1  amd64  Zend OpCache module for PHP\nrc  php5.5-readline  5.5.38-4+deb.sury.org~xenial+1  amd64  readline module for PHP\n<span class=\"regular-prompt\">[<span class=\"user\">chris@home</span> <span class=\"path\">~</span>]$</span> <span class=\"user-input\"></span></pre>\n<p>The <code>dpkg</code> command is Ubuntu’s package manager. The <code>apt</code>, <code>aptitude</code>, <code>apt-get</code>, and other <code>apt-*</code> commands are simply wrappers around <code>dpkg</code>. Calling <code>dpkg --list</code> creates a listing of all the packages on the system. Filtering the output by piping it through <code>grep ^rc</code> results in only showing the removed but unpurged packages (the <code>^rc</code> pattern means that only lines beginning with “rc” should be output). The “r” in “rc” means that the package was removed while the “c” means that the config files remain.</p>\n<p>We can use <code>awk</code> to get a listing of just the package names:</p>\n<pre class=\"terminal notranslate\"><span class=\"regular-prompt\">[<span class=\"user\">chris@home</span> <span class=\"path\">~</span>]$</span> <span class=\"user-input\">dpkg --list | grep ^rc | awk '{ print $2; }'</span>\nphp5.5-cgi\nphp5.5-cli\nphp5.5-common\nphp5.5-json\nphp5.5-opcache\nphp5.5-readline\n<span class=\"regular-prompt\">[<span class=\"user\">chris@home</span> <span class=\"path\">~</span>]$</span> <span class=\"user-input\"></span></pre>\n<p>The <code>awk</code> command processes and manipulates string. By default, when <code>awk</code> reads each line of input, it will separate the input into a series of fields separated by runs of whitespace. You can refer to a specific field by using <code>$NUMBER</code>. In this instance, we tell it to print the second field by using <code>{ print $2; }</code>. You should see the following text:</p>\n<pre class=\"config\">[Element PCM]\nswitch = mute\nvolume = merge\noverride-map.1 = all \noverride-map.2 = all-left,all-right</pre>\n<p>Update this section of text to look like the following (changes are in bold):</p>\n<pre class=\"config\">[Element PCM]\nswitch = mute\nvolume = ignore\nvolume-limit = 0.01\noverride-map.1 = all \noverride-map.2 = all-left,all-right</pre>\n<p>Note that the value <code>0.01</code> can be adjusted as needed to change how quiet and loud the volume is. Making the number smaller reduces the max volume while making the number larger increases the max volume. I tested out <code>0.05</code> and found that the max volume was much louder than I would ever use. I also decided that <code>0.01</code> was technically louder than I’d ever use. I ended up with a value of <code>0.0075</code> (<code>0.005</code> was too quiet) which I felt gave a good maximum volume and resulted in better overall control over the volume range.</p>\n<p>Once you have saved the changes to the file, a couple things still need to be done in order to have this change go into effect.</p>\n<p>Inside your terminal window (it should be behind the text editor), run the following command:</p>\n<pre class=\"terminal notranslate\"><span class=\"regular-prompt\">[<span class=\"user\">chris@home</span> <span class=\"path\">~</span>]$</span> <span class=\"user-input\">pulseaudio -k</span></pre>\n<p>This will force the audio system in Ubuntu to reload. If you had any application that was playing audio, it will go silent at this point. Any application that is now silent will have to be closed and reopened.</p>\n<p>You can now test out the change to see if you need to adjust the number down or up to make the volume to your liking. Remember to save the file and run <code>pulseaudio -k</code> each time you change the number.</p>\n<h2>What Didn’t Work</h2>\n<p>I found a number of guides that recommended that the <code>/etc/pulse/default.pa</code> file be modified and have the following lines added:</p>\n<pre class=\"config\">load-module module-alsa-sink control=PCM\nload-module module-udev-detect ignore_dB=1</pre>\n<p>Neither of these worked for me. The first config change had no effect. The second config change did fix the issue with the sound muting at low volumes, but the volume was still far too loud with just 1% volume being what I consider to be an 80-90% volume level.</p>\n<h2>Technical Details</h2>\n<p>For those that are interested in the technical details of this USB device, the following information is for you.</p>\n<h3><code>lsusb</code></h3>\n<p>Full verbose details (<code>lsusb -vvvv</code>) can be found <a href=\"https://chrisjean.com/files/sabrent-usb-audio-lsusb-vvvv.txt\">here</a>.
